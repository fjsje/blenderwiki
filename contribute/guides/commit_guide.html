提交指南

Access to directly submit changes is limited to people with commit access to the repository. Once you are provided with commit access you can start committing directly instead of creating a patch file.

你可以从你的 Git 客户端或使用 Git 命令行工具来提交 commit。以下命令将会创建一个 commit 并将它推送到代码仓库:

git commit -m "This is what I did"
git push

如果你省略了 -m "message"，将提示你在文本编辑器中输入留言。

Tip

提交之前，您应该确保始终使用最新的修订版本。如果最新的修订版本中有冲突的更改，您可能无法直接提交。

To avoid this update your local repository before committing (run make update).

写一个好的承诺信息

当对手册进行与 Blender 中的特定提交（更改）直接相关的更改时，将提交标题与提交给 Blender 的标题相同很有帮助。要求您包含对 Blender 源代码所做的提交的提交哈希。

例如，提交 rBM8473 包括指示与散列 rBa71d2b260170 一起做出的改变的描述性说明。可以从文档任务中为即将发布的特定版本提供的 URL 中提取哈希。

其他更一般的变更不必遵循上述政策，但仍有必要明确说明您所做的变更及其原因。当您分别进行常规清理或修复时，使用前缀词（如 Cleanup: 或 Fix:）来前缀提交标题可能会有所帮助。

编写好的提交消息有助于管理员跟踪所做的更改，并确保所有新功能都得到正确记录。
