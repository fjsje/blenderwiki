演示模式

待补充。

激活

  • 打开 Blender 并转到 “偏好设置”，然后转到 “插件” 选项卡。

  • 选择 “系统” 类别，然后勾选 “演示模式” 以启用插件。

配置

这是从文件菜单中的 文件 ‣ 演示模式(设置) 完成的，您可以在此处选择递归搜索要循环的 blend 文件的目录。

此时，可以选择如何处理每个文件，主要选项是渲染或播放动画，其中包含播放多少个循环、渲染完成后等待的时间等选项。

自动配置

自动 选项根据窗口布局使用图像/节点编辑器的情况，在 渲染 和 播放 选项之间进行选择。如果您愿意在所有文件上使用这些设置，则可以启用 立即运行 选项并确认。

修改配置

如果要为每个文件设置选项，您可以在设置期间禁用 立即运行，然后选择 demo.py 文本数据块。此文件每个文件有一行，可以使用 blend 文件编辑和保存设置。当 demo.py 存在时，您只需转到 文件 ‣ 演示模式(开始)。

属性

这些关键参数显示在生成的 demo.py 中，例如：

dict(anim_cycles=2, anim_render=False, anim_screen_switch=0.0, anim_time_max=8.0, anim_time_min=4.0,
display_render=4.0, file='foobar.blend', mode='AUTO'),

  • anim_cycles -- 动画播放次数。

  • anim_render -- 渲染整个动画（仅适用于 mode='RENDER'）。

  • anim_screen_switch -- 切换屏幕之间的时间（以秒为单位）或 0 以禁用。

  • anim_time_max -- 显示动画的最大秒数（如果最终帧无缘无故非常高）。

  • anim_time_min -- 显示动画的最小秒数（用于小循环）。

  • display_render -- 在继续之前显示渲染图像的时间（以秒为单位）。

  • file -- blend 文件的文件路径。

  • mode -- AUTO / PLAY / RENDER -- 加载时执行哪些动作。

便携化配置

设置 demo.py 后，您可能需要将其移至其他系统。在这种情况下，路径可能不匹配，如果匹配，请更新 search_path 变量以指向新系统上的路径。您还可以设置 search_path 以使用当前 blend 文件的目录:

search_path = "//"

用法

演示启动后，它将在同一 Blender 实例中播放、渲染和加载不同的文件。自使用模态运算符以来，您仍然可以在演示运行时与 Blender 一起使用。

如果要停止演示模式，可以按 Esc 并继续使用 Blender 。要再次启用，文件菜单右侧有三个按钮可导航 上一个 / 暂停 / 下一个。

局限

  • 一个文件不能播放动画然后渲染，您需要复制 demo.py 中的行，并将一个模式设置为 RENDER，另一个设置为 PLAY。
